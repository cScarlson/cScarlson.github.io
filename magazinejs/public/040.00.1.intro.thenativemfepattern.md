
The Native MFE Pattern
================================================================

## Myth 1
_You need a heavy framework like RSBuild, Module Federation, or similar to build a Micro Frontend Architecture_.

## Myth 2
_HTML and Web Technologies do not give us simple partials/includes_.

## Reason
I invented _The Native MFE Pattern_, turning The Web into a Plugin Architecture.
- You don't need an MFE Framework.
- The Web always had partials/includes.
- All you need is a webpage, GitHub Pages site, S3 Buckets, or any other CI/CD system.

If you've made it this far, give yourself a big pat on the back for being a glutton for punishment, like me. The last chapter in this book demystifies a couple of misconceptions which, after inventing this pattern, I realized is somewhat of a big deal.

The Native MFE Pattern does not simply advance one language or one capability for specific use-cases; in advances the state of The Web, it turns the entire web infrastucture into a Plugin Architecture in a way that any website or webapp can consume fully-capable components and partials from trusted sites, even if they are simply hosted one's GitHub Pages.
