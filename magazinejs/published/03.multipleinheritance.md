
Multiple Inheritance in JavaScript
================
Yes, it's possible. No, Multiple Inheritance is not _evil_. It's actually where Mixins thrive. Yes, it's been in JavaScript for just about ever. In fact, it's one of the only other ways to do inheritance in JavaScript beside Prototypal or mixins with the spread operator.

## Multiple Inheritance and Mixins
```javascript
const ParentClassA = function ParentClassA() {
    ...
    // export precepts
    this.a = true;
    
    return this;
};

const ParentClassB = function ParentClassB() {
    ...
    // export precepts
    this.b = true;
    
    return this;
};

const ParentClassC = function ParentClassC() {
    ...
    // export precepts
    this.c = true;
    
    return this;
};

const ChildClass = function ChildClass() {
    ...
    // export precepts
    ParentClassA.apply(this);
    ParentClassB.apply(this);
    ParentClassC.apply(this);
    this.abc = true;
    
    return this;
};

const child = new ChildClass();

console.log(child);
// > { a: true, b: true, c: true, abc: true }
```

## Synopsis
The lesson should be: _don't fear away from part of the language when it's more useful than others_.
