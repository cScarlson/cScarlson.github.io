
I18n/L10n
================

Internationalization is easy. We basically just need 2 things...

## Participants
- `{lang}.locale.json` files: translation texts.
- `translate(key)`: a function to perform translation.
- Interjectors (optional): faculties for performing interpolations, formatting, and other processing.

## `{lang}.locale.json`
You'll need one or more files for storing static translation texts.

### Examples

##### `en.locale.json`
```json
{
    "PURCHASE:COMPLETE:MESSAGE": "Thank you for purchasing your new {vehicle}!",
    "TAXON:VEHICLE:AUTO": "car",
    "TAXON:VEHICLE:MOTO": "motorcycle",
    "USER:SETTINGS:PRIVACY:BUTTON:SAVE": "Save",
    "USER:SETTINGS:PRIVACY:BUTTON:CANCEL": "Cancel",
    "USER:SETTINGS:PRIVACY:TOA": "{readme}"
}
```

##### `es.locale.json`
```json
{
    "PURCHASE:COMPLETE:MESSAGE": "¡Gracias por comprar su nuevo {vehicle}!",
    "TAXON:VEHICLE:AUTO": "auto",
    "TAXON:VEHICLE:MOTO": "motocicleta",
    "USER:SETTINGS:PRIVACY:BUTTON:SAVE": "Ahorrar",
    "USER:SETTINGS:PRIVACY:BUTTON:CANCEL": "Cancelar",
    "USER:SETTINGS:PRIVACY:TOA": "{readme}"
}
```

### A Word on Conventions

#### Values
It is ok to use best-guess on case-sensitivity/capitialization but try to leverage CSS `text-transform` in favor of the locale files whenever possible.

#### Keys
Notice that we used colon:delimitation for our key-names. It is _highly_ recommended _not_ to leverage _nesting_ of object structures within the locale files. We want to keep the structures _flat_ as it will keep our code much [much] simpler.


## Translate Function
The most basic way is to write a single function that takes in a key and looks up a translation. However, that gives us less leverage to control the configuration of our locale variable. That said, we'll export a class that allows us to configure our language while it is driven by multiple mechanisms. Here it is in a nutshell...

### Stronger Coupling (without Injerjectors)
```typescript
import { locales } from '**/locales';

const { language } = navigator;

export default (key: string) => locales[language][key];
```

### Weaker Coupling (with Injerjectors)
```typescript
import { locales } from '**/locales';
import { interject } from '**/interjectors';

const { LANG } = env;
const { language } = navigator;
const { search } = location;
const params = new URLSearchParams(search);
const lang = params.get('lang');
const locale = lang || language || LANG;

export default new (class Translator {
    locale: object = locales[this.lang];
    
    constructor(public lang: string) {}
    
    translate = (key: string) => {
        const { locale } = this;
        const { [key]: text } = locale;
        const translation = interject({ key, text, locale });
        
        return translation;
    };
    
})(locale);  // <-- configuration
```

## Interjectors (Optional)
Oftentimes you'll want to perform some string interpolation on the text translations before they are displayed. You may have placeholders, formatting concerns, or even temporary hotfixes. Here's how we can rig that up...

```typescript
import { locales } from '**/locales';
import { default as toa } from 'text!**/toa.md';

type TDetails = { key: string, text: string, locale: string };
type TInterjector = (details: TDetails) => TDetails;

const interjectors: Set<TInterjector> = new Set()
    .add(interpolateTermsOfAgreement)
    .add(interpolateVehicle)
    ;

function interpolateTermsOfAgreement({ key, text, locale }) {
    if (key !== 'USER:SETTINGS:PRIVACY:TOA') return { key, text, locale };
    return { key, text: markdown.parse(toa), locale };
}

function interpolateVehicle({ key, text, locale }) {
    if (key !== 'PURCHASE:COMPLETE:MESSAGE') return { key, text };
    const { ['TAXON:VEHICLE:AUTO']: taxon } = locale;
    const translation = utils.interpolate(text)({ vehicle: taxon });
    
    return { key, text: translation, locale };
}

export function interject(details) {
    const { text: translation } = [ ...interjector ].reduce( (details, fn) => fn(details), details );
    return translation;
}
```

## Synopsis
This was incredibly simple. All we basically did is take write a function that takes a key and looks up the translation from a locale file. We expanded our translate function to run the lookup value through some _interjectors_ so that we could target specific keys and perform any amount of magic we want in any case we'd like. Ultimately, our interjectors system implements _The Chain Of Responsibility Pattern_, which makes our lives easier.
