
A Reactive Datastore
================
This example assumes you know all you need about _The Observer Pattern_. The Observer Pattern is a _very_ simple and small pattern that can be especially helpful if all you need is a simple Hub-N-Spoke Model of subscribers pointed at something that maintains a state of interest.

##### `**/core/store/store.ts`
```typescript
import { reducer } from './reducer';

class Store extends Subject {
    private state = {};
    
    constructor(initial) {
        this.state = initial;
    }
    
    dispatch(action: { type: string, payload?: any }) {
        const { state } = this;
        const reduced = reducer.execute(state, action);
        
        this.state = reduced;
        this.notify(reduced);
        
        return this;
    }
    
}
```

##### `**/core/store/reducer.ts`
```typescript
class Reducer {
    
    constructor(options) { ... }
    
    execute(state, action) {
        const { type, payload } = action;
        if (type in this) return this[type](state, action);
        return state;
    }
    
    ['USER:SETTINGS:CHANGED'](state, action) {
        const { payload } = action;
        const { data } = payload;
        const { settings } = data;
        
        return {
            ...state,
            user: {
                ...state.user,
                settings: { ...state.user.settings, ...settings }
            }
        };
    }
    
}
```

##### `**/some.component.ts`
```typescript
const component = new (class SomeComponent extends Framework {
    private settings = { };
    
    constructor(private store: Store) {
        store.attach(this);
    }
    
    update(state) {
        this.settings = state.user.settings;
        this.rerender(this);
    }
    
    onDestroy() {
        this.store.detach(this);
    }
    
    @delegate('change')
    handleUserSettingsChange(e, settings) {
        this.store.dispatch({ type: 'USER:SETTINGS:CHANGED', payload: settings });
    }
    
})();
```

## Synopsis
We effectively just created our own Redux/Context/NGRX state store in only about 21 LOC (lines of code). We excluded the LOC length for The Observer Pattern's _Subject_, although, that also only looks like about 20 lines of code long. Moreover, we can simply wrap our Store with, say, a "`StoreDecorator`" and _interject_ on any invocations of `dispatch` so that we can hit a server before we invoke the store's dispatch method and hit the reducer (see "Why you don't need Redux/Sagas/Thunk").
